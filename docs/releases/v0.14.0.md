# v0.14.0

## Release Summary

This release is a major step forward for the forum-topics fork.

It hardens Telegram delivery under real group-topic pressure, redesigns streaming so in-progress updates behave more like upstream while still respecting fork-specific topic workflows, improves pinned/context state consistency, and closes several rough edges around compaction, TTS, and final message delivery.

## Highlights

- Reworks streamed assistant delivery into a durable shared stream lane with topic-safe rollover at Telegram message limits
- Sends file mutation results as attachments without duplicating `write`, `edit`, or `apply_patch` text in the stream
- Delivers the final assistant reply separately with proper Markdown formatting and final-only TTS support
- Makes Telegram `retry_after` handling much more resilient across stream edits, file sends, pinned updates, topic title sync, and audio delivery
- Keeps pinned context, keyboard context, and post-compaction state aligned more reliably across long-running sessions

## Notes

- This release aligns the fork version with upstream `v0.14.0`; there were no newer upstream commits beyond the already reviewed sync head in `FORK_SYNC.md`
- Forum-topic streaming remains fork-specific: streamed messages stay visible after completion instead of being deleted like upstream interim messages
- File attachments act as intentional visible boundaries in the live stream

## Included Tests

- Live stream lane ordering, rollover, final cleanup, and clear/reset behavior
- Telegram rate-limit retry handling for text, files, pinned updates, and TTS sends
- Pinned/context reload behavior after compaction and scoped context clearing
- Final assistant delivery, compaction notices, and scoped context refresh behavior
