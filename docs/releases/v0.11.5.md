# v0.11.5

## Release Summary

This release tightens topic-scoped message delivery and adds a lightweight way to recover the latest visible session turn from Telegram.

The biggest user-facing improvement is that question flows, completion summaries, and related bot replies stay anchored to the correct forum topic more consistently. It also adds `/last`, which lets you fetch the latest visible user or assistant message from the active session without leaving Telegram.

## Highlights

- Adds `/last` to show the latest visible turn from the current session
- Keeps poll completion and other session-scoped replies in the owning Telegram topic
- Improves startup and initial-session feedback around new session flows
- Expands tests for topic routing, session previews, and DM help command behavior

## Included Since v0.11.4

- Shared session-history helpers for loading visible conversation turns
- New `/last` command with topic-aware reply routing and error handling
- Stronger topic-thread delivery for question completion summaries and related replies
- DM help output aligned with the allowed command set
- Regression coverage for topic-scoped question flows and latest-message previews

## Stability Notes

- Topic-bound sessions continue to route interactive follow-ups back into the originating thread
- Session previews now use the bound session directory instead of assuming the current project worktree
- The release remains backward-compatible with existing group-topic session bindings and control flows
