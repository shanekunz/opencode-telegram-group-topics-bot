# v0.15.3

## Release Summary

This patch release bundles every fork change since the last published release `v0.15.1`, including the unpublished `v0.15.2` stream-ordering work and a new `/tasklist` fix for large scheduled-task prompts.

It tightens final-message delivery ordering in Telegram, keeps `apply_patch` attachments scoped to the right files, prevents scheduled-task listings from failing when prompts get too large, and adds local Remotion-based tooling for release and video-production workflows.

## Highlights

- Preserves event and delivery ordering through final assistant completion handling, including queued completions and post-final cleanup
- Keeps `apply_patch` attachment output scoped to the correct file paths
- Fixes `/tasklist` failures caused by Telegram's message-length limit by truncating long prompt previews
- Adds Remotion, React, and related dev tooling for local media and release-production workflows
- Ignores local AI-tooling and video-production workspace artifacts in git

## Notes

- This release supersedes the unpublished `v0.15.2` release-prep commit and includes all fixes landed after `v0.15.1`
- The new Remotion-related packages are development-only dependencies and do not change the bot's runtime dependencies
- No bot configuration changes are required for the stream-ordering or `/tasklist` fixes

## Included Tests

- Full project build validation with `npm run build`
- Full ESLint validation with `npm run lint`
- Full automated test suite with `npm test`
- Targeted `/tasklist` regression coverage for long prompt truncation
