# v0.15.1

## Release Summary

This release brings the fork forward through upstream `v0.15.0` and adds fork-specific reliability fixes that matter in real Telegram group-topic usage.

It ports the upstream runtime and UI updates, tightens agent fallback and live tool-stream behavior, and hardens scheduled-task execution so unattended runs recover more reliably after transient failures or bot restarts.

## Highlights

- Ports upstream `v0.15.0` runtime updates into the group-topics fork
- Keeps agent selection aligned with project availability, including callback-driven changes
- Improves live tool-stream rendering for todo/subtask updates and long bash command output
- Carries the current git branch into pinned project metadata when it can be resolved
- Hardens scheduled-task execution with explicit unattended permissions, timeout-safe transport settings, stale-running recovery, and better delivery failure handling

## Notes

- This fork release includes upstream `v0.15.0` changes plus additional fork-only fixes landed afterward
- Git branch display is part of pinned project metadata rather than the `/status` command output
- Scheduled-task behavior remains fork-specific where it depends on Telegram topic delivery and unattended execution policy

## Included Tests

- Agent selection fallback when a requested agent is no longer available for the current project
- Live-stream replacement behavior for replace-keyed service updates
- Scheduled-task executor behavior for prompt success, prompt failure, and empty assistant responses
- Scheduled-task runtime recovery for stale `running` state and delivery failures after execution
