# v0.13.4

## Release Summary

This patch release smooths out the new scheduled-task and queueing UX in Telegram.

It clarifies that scheduled tasks inherit the current `🎛️ Session Control` agent and model defaults at creation time, keeps legacy scheduled topics renamed to the new output lane, and makes queued prompts announce themselves when they begin processing.

## Highlights

- Clarifies scheduled-task creation UX around inherited agent and model defaults
- Renames reused legacy scheduled topics to `⏰ Scheduled Task Output`
- Blocks most slash commands inside scheduled output topics with clearer guidance back to `🎛️ Session Control`
- Announces the next queued prompt when it starts and shows a preview of the queued message
- Leaves a maintenance note in code to prefer a native OpenCode headless queue if one is added later

## Notes

- Queueing is still implemented in the Telegram bot layer; upstream bot flow does not yet expose a native queue
- Scheduled output topics remain delivery-only lanes, not active coding lanes
- Existing legacy scheduled topics are renamed when the bot next reuses them for that project

## Included Tests

- Scheduled topic reuse and legacy rename behavior
- Scheduled output command blocking
- Queued prompt start notifications and previews
