# v0.13.3

## Release Summary

This patch release tightens the Telegram topic UX around scheduled task output and improves recovery when you return to an existing OpenCode session from outside Telegram.

It also adds Telegram-side prompt queuing for busy sessions, so follow-up messages can wait their turn instead of being dropped.

## Highlights

- Renames the General topic to `🎛️ Session Control` for clearer separation from scheduled output
- Renames forum scheduled topics to `⏰ Scheduled Task Output`
- Blocks normal prompts inside scheduled output topics with clearer guidance back to session control
- Makes `/last` prefer the most recent assistant reply instead of only the latest visible user turn
- Queues follow-up prompts per session, then announces and starts the next queued message automatically after the current run finishes

## Notes

- Queueing is implemented in the Telegram bot layer; upstream still checks busy state and does not queue prompts in the bot flow
- Queued prompts stay tied to the same session and thread, and they start automatically on `session.idle`
- When a queued prompt starts, Telegram now posts a short "starting next queued message" notice with the queued prompt preview
- Scheduled task output topics remain delivery-only lanes, not active coding lanes

## Included Tests

- Scheduled topic naming and blocking behavior
- `/last` assistant-response preference
- Prompt queueing and automatic dispatch after idle
