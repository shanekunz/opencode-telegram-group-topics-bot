# v0.15.2

## Release Summary

This patch release fixes a delivery-ordering bug in the OpenCode event stream that could let the final assistant response appear before earlier tool updates or file-edit attachments finished rendering in Telegram.

The bot now preserves upstream SSE event order through callback processing, so final responses no longer cut ahead of still-valid in-flight messages from the same run.

## Highlights

- Fixes out-of-order event delivery in the OpenCode stream worker
- Prevents final assistant messages from jumping ahead of tool-call updates and file attachments
- Preserves real intermediate output instead of suppressing late-but-valid messages

## Notes

- The fix is implemented in the event subscription pipeline rather than by filtering bot output after the fact
- Per-session delivery behavior remains serialized by the existing Telegram delivery queue
- No user-facing command or configuration changes are required for this release

## Included Tests

- Full project build and lint validation
- Full automated test suite run after the event-ordering change
